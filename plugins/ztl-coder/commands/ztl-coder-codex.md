# Codex 第二意见

使用 OpenAI Codex 提供独立的第二意见，交叉验证关键决策。

## 使用场景

- 重要架构决策
- 复杂代码审查
- 安全敏感修改
- 算法正确性验证
- 多方案选择困难

## 工作流程

### Step 1: 收集上下文

自动收集相关上下文：

```bash
# 当前分支信息
git branch --show-current
git log -5 --oneline

# 相关文件
git diff --name-only main...HEAD

# 问题描述
# 从用户输入或当前任务获取
```

### Step 2: 构建提示词

```markdown
# Codex 第二意见请求

## 背景
{project_context}

## 问题
{问题描述}

## 当前方案
{当前代码或方案}

## 请评估
1. 这个方案是否正确？
2. 有没有潜在问题？
3. 有没有更好的替代方案？
4. 有什么建议？
```

### Step 3: 调用 Codex

使用 OpenAI API 获取独立意见。

### Step 4: 对比分析

将 Codex 意见与当前方案对比：

```markdown
# 第二意见报告

## 原始方案
{原始方案摘要}

## Codex 意见
{Codex 意见摘要}

## 对比分析

| 维度 | 原始方案 | Codex 意见 | 一致性 |
|------|----------|------------|--------|
| 正确性 | {评估} | {评估} | ✅/⚠️ |
| 安全性 | {评估} | {评估} | ✅/⚠️ |
| 性能 | {评估} | {评估} | ✅/⚠️ |
| 可维护性 | {评估} | {评估} | ✅/⚠️ |

## 分歧点

### Codex 指出的问题
1. {问题1}
2. {问题2}

### Codex 建议的改进
1. {建议1}
2. {建议2}

## 最终建议

{综合两个意见的建议}
```

## 使用示例

### 架构决策验证

```
用户: 我们要用 Redis 做缓存，帮我验证这个决策

Codex:
1. 收集项目上下文（技术栈、规模、需求）
2. 调用 Codex 评估 Redis 选型
3. 对比分析优劣
4. 提供最终建议
```

### 代码审查交叉验证

```
用户: 审查这段认证代码，要确保安全

Codex:
1. 收集认证相关代码
2. 调用 Codex 独立审查
3. 对比两个审查结果
4. 综合安全建议
```

### 算法正确性验证

```
用户: 这个排序算法在大数据量下会不会有问题？

Codex:
1. 分析算法实现
2. 调用 Codex 评估复杂度
3. 对比性能预期
4. 提供优化建议
```

## 配置选项

### API 配置

```json
{
  "codex": {
    "model": "gpt-4",
    "temperature": 0.3,
    "maxTokens": 2000,
    "timeout": 30000
  }
}
```

### 触发条件

自动触发条件（可配置）：

- 涉及安全敏感代码
- 修改核心架构
- 性能关键路径
- 用户明确请求

## 注意事项

- Codex 意见仅供参考
- 最终决策由用户做出
- 敏感信息不应发送到外部 API
- 建议在非敏感场景使用

## 隐私保护

默认行为：
- 不发送完整代码
- 仅发送必要上下文
- 脱敏敏感信息
- 本地缓存结果

可配置：
```json
{
  "codex": {
    "privacy": {
      "stripSecrets": true,
      "anonymizeNames": false,
      "maxCodeLength": 1000
    }
  }
}
```
