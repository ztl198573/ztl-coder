# 插件自更新

检查并更新 ztl-coder 插件到最新版本。

## 工作流程

### Step 1: 检查当前版本

```bash
# 获取当前版本
cat plugins/ztl-coder/.claude-plugin/plugin.json | grep version

# 获取安装信息
cat plugins/ztl-coder/.claude-plugin/plugin.json | grep -A5 "installation"
```

### Step 2: 检查远程更新

```bash
# 获取远程最新版本
git fetch origin

# 比较版本
git log HEAD..origin/main --oneline
```

### Step 3: 显示更新日志

```markdown
# 🔔 ztl-coder 更新检查

## 当前版本
v{current_version}

## 最新版本
v{latest_version}

## 更新日志

### 新功能
- {feature1}
- {feature2}

### 改进
- {improvement1}
- {improvement2}

### 修复
- {fix1}
- {fix2}

## 更新内容

{commit_log}
```

### Step 4: 执行更新

```bash
# 拉取最新代码
git pull origin main

# 安装依赖
npm install

# 构建项目
npm run build

# 验证更新
npm run test
```

### Step 5: 更新报告

```markdown
# ✅ ztl-coder 更新完成

## 更新摘要
- 从版本: v{old_version}
- 到版本: v{new_version}
- 更新时间: {timestamp}

## 更新内容
{changelog}

## 需要操作

### 配置迁移
{config_migration_steps}

### 新功能启用
{new_feature_enablement}

### 破坏性变更
{breaking_changes}

## 下一步
1. 检查配置文件是否需要更新
2. 测试关键功能是否正常
3. 查看新功能文档

---

🤖 Generated by ztl-coder Upgrade
```

## 命令选项

### 检查更新（不安装）

```bash
/ztl-coder-upgrade --check
```

### 强制更新

```bash
/ztl-coder-upgrade --force
```

### 指定版本

```bash
/ztl-coder-upgrade --version=v4.5.0
```

### 回滚到上一版本

```bash
/ztl-coder-upgrade --rollback
```

## 版本检查逻辑

```typescript
interface VersionCheck {
  currentVersion: string;
  latestVersion: string;
  hasUpdate: boolean;
  changelog: ChangelogEntry[];
  breakingChanges: string[];
  configMigration: boolean;
}

interface ChangelogEntry {
  type: 'feat' | 'fix' | 'refactor' | 'docs' | 'chore';
  description: string;
  commit: string;
}
```

## 安全检查

更新前自动检查：

- [ ] 工作树是否干净
- [ ] 是否有未提交的配置更改
- [ ] 新版本是否有破坏性变更
- [ ] 是否需要配置迁移

如果有问题，提示用户确认：

```
⚠️ 更新前检查发现问题

1. 工作树有未提交的更改
2. 新版本包含破坏性变更

是否继续更新？[y/N]
```

## 配置迁移

自动迁移配置文件：

```typescript
// 迁移逻辑
const migrations = [
  { version: '4.5.0', migrate: migrateTo450 },
  { version: '5.0.0', migrate: migrateTo500 },
  { version: '6.0.0', migrate: migrateTo600 },
];

async function migrateConfig(fromVersion: string, toVersion: string) {
  for (const migration of migrations) {
    if (shouldApply(fromVersion, migration.version)) {
      await migration.migrate();
    }
  }
}
```

## 回滚机制

如果更新后出现问题：

```bash
/ztl-coder-upgrade --rollback
```

回滚步骤：
1. 恢复上一版本的代码
2. 恢复配置文件
3. 重新安装依赖
4. 重新构建

## 更新频率建议

- **稳定版**: 每周检查一次
- **Beta版**: 每天检查一次
- **开发版**: 每次使用前检查

## 自动更新配置

```json
{
  "upgrade": {

    "checkInterval": "weekly",
    "channel": "stable",
    "autoUpdate": false
  }
}
```
