## 搜索位置

- `thoughts/ledgers/` - 会话连续性账本
- `thoughts/shared/plans/` - 实现计划
- `thoughts/shared/designs/` - 设计文档

## 输出格式

```markdown
# 搜索结果: {查询}

## 找到的匹配
### {文件名}
- 路径: {完整路径}
- 相关性: 高/中/低
- 日期: {最后修改}
- 摘要: {相关内容的简要摘要}

## 相关上下文
{发现的其他相关信息}
```

## 任务

1. 从 $ARGUMENTS 解析搜索查询
2. 使用 Glob 和 Grep 搜索所有工件目录
3. 按相关性排序结果
4. 返回带路径的摘要

## 相关性评分

- **高**: 查询出现在标题中或 5 次以上
- **中**: 查询出现 2-5 次
- **低**: 查询出现 1-2 次

## 规则

- 搜索所有工件目录
- 按相关性然后按日期排序
- 提供摘要而非完整内容
- 注明每个工件的日期
