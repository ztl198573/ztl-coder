# 周回顾（Retrospective）

执行结构化的周回顾，分析团队感知、识别改进机会、规划下周行动。

## 回顾框架

### 感知分析维度

1. **生产力（Productivity）**
   - 本周完成了什么？
   - 有什么阻碍了进度？
   - 如何提高效率？

2. **质量（Quality）**
   - 交付质量如何？
   - 有哪些技术债务？
   - 如何改进质量？

3. **协作（Collaboration）**
   - 团队协作是否顺畅？
   - 沟通是否有效？
   - 如何改善协作？

4. **学习（Learning）**
   - 学到了什么新东西？
   - 有什么技能提升？
   - 需要什么培训？

5. **士气（Morale）**
   - 团队士气如何？
   - 有什么压力来源？
   - 如何提升士气？

## 执行步骤

### Step 1: 收集本周数据

```bash
# 获取本周提交
git log --since="1 week ago" --oneline --author="$(git config user.name)"

# 获取本周 PR
gh pr list --state merged --search "merged:>=2024-01-01"

# 获取本周 Issues
gh issue list --state closed --search "closed:>=2024-01-01"

# 检查 TODO 变化
git diff HEAD@{1.week.ago} -- "**/TODO*" "**/TODOS*"
```

### Step 2: 分析各维度

对每个维度进行评估：

```markdown
## 生产力分析

### 完成项
- {item1}
- {item2}

### 阻碍因素
- {blocker1}
- {blocker2}

### 效率评估
- 评分: X/10
- 改进建议: {suggestion}
```

## 使用场景

- 每周五下午

## 注意事项

- 保持诚实和客观
- 关注可执行的改进

- 庆祝成功，学习失败
